> [!NOTE] Note
> This is aggregate feedback from playtests of version 0.8.0

See: [[2026-07-23-ring-1-maturity-audit#Maturity inventory|Maturity Inventory]]

| Ring 1 domain                                                   | Feedback                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| --------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Generate and start a world                                      | Generally OK. Main issues are with the interface UX, not with the functionality. <br>- The game should start with a random seed already entered, not 0.<br>- I want to be able to enter a text string as a seed<br>- If there's a length limit on seed manual entry it should clamp to that length, and not let me enter more text than allowed                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Bootstrap population and survive seasonal Energy pressure       | Needs improvement. Energy pressure is very weak right now and also hard to visualize and understand. This is a complex issue complicated in part by the player not having a need to build the population and thus the energy needs are never unmanageable to begin with. The seasonal energy changes are also difficult to notice because we only present it in places where the player isn't trying to use the energy. The seasonal effect is also a multi-tick horizon, and it's also very hard for the player to see and understand the state of the game over multiple ticks. We only present the current snapshot in most cases.                                                                                                                                                                                                                                                                                 |
| Inspect stocks, developments, commitments, and limiting reasons | Acceptable for the quality level we're aiming for at this phase of development                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Bank or develop physical margin                                 | This is always accomplished by building more collectors and batteries                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Construct and operate Shipyards                                 | Acceptable at this stage. It's not clear how to manage multiple shipyards, but it's also barely useful to the player to build multiple shipyards on a system.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Scout, travel, observe, and receive delayed reports             | Needs improvement. The delayed information exists at a technical level, but it's very hard for the player to see or understand that this is happening. The game is already tick-based, and many things change per-turn. Right now there's no way for the player to know that a probe arrived somewhere, but that the information is on the way back. To a player, this can seem like the same thing as the probe simply taking longer to arrive and the information updating instantly on arrival.<br>We're also tracking a mechanical implementation issue with how the expansion frontier discovery is implemented in TODO-001 that will improve the scouting mechanic.                                                                                                                                                                                                                                             |
| Prepare, launch, and resolve an expedition                      | Needs improvement. The UX for selecting where the expeditions initial developments are placed is extremely confusing. Right now there is also no strategy required from the player other than avoiding filling up slots that are on bodies with resources.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Found and unlock a remote system after delayed outcome receipt  | Needs improvement. Feedback is the same as the issues with the scout information and the feedback of the previous item.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Continue governing origin and founded systems through the TUI   | Acceptable at this stage. After other parts of the core loop are improved, this will need to be revisited. Systems are able to extract all of their resources rather quickly. e.g. if there's 400 ore and 4 slots, you could build 4 extractors there. After a system extracts all the ore and uses it all up, all it can do is generate and consume energy.<br><br>This is OK for now and expected.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Understand the complete cycle as one repeated strategy loop     | Moving in the right direction. The basic idea of the core loop is present and the game is playable, but gets stale quickly. On the development side this is advantageous to us so we can iterate quickly before playtesting takes much too long and understanding the game is much harder.<br><br>I want to call out the need to _communicate_ core mechanics more clearly to the player. Part of this will be polishing the presentation of mechanical ideas, the fiction, and fantasy so that the game loop _feels good_. We don't always need to make the game mechanically or strategically complex, as long as what the mechanics and strategy communicate is rich and expansive.<br><br>Finding the right places to present TUI _visuals_ like graphs, bars, charts, images will be very powerful. Just because this game is TUI does not mean that text is the best way to communicate all of the information. |
